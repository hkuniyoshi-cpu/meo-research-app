# MEO調査アプリ 設計ドキュメント

- 作成日: 2026-06-19
- ステータス: 設計合意（実装計画前）
- 運営: SearchMania Inc.

## 1. 目的とコンセプト

事業名と住所（またはエリア名）を入力するだけで、対象店舗の **Googleマップ整備度** と **検索評価（想定）** を即座に診断するWebアプリ。

2つの用途を同等に満たす：
1. **自店診断** — 店舗オーナーが自店のMEO整備状況を把握し、改善ポイントを知る
2. **競合調査** — 競合店の整備度・口コミ・評価をサッと比較する（選択制）

無料診断をフックに、最終的に SearchMania の有料サービスへ誘導するリード獲得ツールでもある。

### 成功基準
- 事業名＋住所のみ（メール不要）で誰でも診断できる（離脱を最小化）
- コストがほぼ0で不特定多数の公開利用に耐える
- 「ちゃんと調査している」体感を演出で担保する
- メールアドレス取得によるリード獲得が機能する

## 2. スコープ

### 含む（MVP）
- 入力（事業名・住所/エリア・競合比較トグル）
- 整備評価（100点満点・客観）
- 検索評価（想定・相対ポジション）
- 競合比較（任意）
- 改善ポイント（無料版は上位3つ＋もったいぶり）
- メール詳細版（確認コードで解錠・整形レポート）
- ボット/スパム対策・レート制限
- SearchManiaへの控えめな誘導

### 含まない（将来拡張 / YAGNI）
- 実順位の地理グリッド計測（ヒートマップ）— コスト高のため想定値で代替
- AI検索（ChatGPT/Gemini等）での表示有無チェック — LLM課金が発生するため将来検討
- ログイン/ダッシュボード/履歴保存
- 多言語対応

## 3. アーキテクチャ（Cloudflareスタック）

```
[ブラウザ/Cloudflare Pages(静的フロント)]
        │ 1. Turnstileトークン付きで診断リクエスト
        ▼
[Cloudflare Worker (API中継・本体ロジック)]
   ├─ Turnstile検証
   ├─ レート制限（IP/日次, KV）
   ├─ KVキャッシュ参照（place_idキー, TTL 7〜30日）
   ├─ Google Places API (New) 呼び出し（キャッシュミス時のみ）
   │     - Text Search: 事業名＋住所/エリア → place_id 特定
   │     - Place Details: 詳細フィールド（Field Maskで限定）
   │     - 競合ON時: Nearby/Text Search で同業近隣N件
   ├─ スコアリング（整備評価・検索評価想定）
   └─ 結果JSON返却（無料版は改善点を一部マスク）
        │
        ▼
[結果画面] ──(メール詳細版希望)──> [Worker: 確認コード発行→Resendでメール送信]
                                         └─ コード手入力で解錠 → 整形レポート
```

### コンポーネント分割
| コンポーネント | 役割 | 依存 |
|---|---|---|
| `frontend` (Pages) | 入力UI・ローディング演出・結果描画 | Worker API, Turnstile |
| `worker-diagnose` | 診断本体（検証→キャッシュ→Places→採点→マスク） | Places API, KV, Turnstile |
| `worker-lead` | メール確認コード発行・検証・詳細版解錠 | KV, Resend |
| `scoring` (Worker内モジュール) | Places応答→整備/検索スコア算出 | なし（純粋関数・単体テスト可） |
| `places-client` | Places API呼び出し＋Field Mask＋正規化 | Places API |
| `cache` | KV読み書き（place_id/結果/コード/レート） | KV |

`scoring` は外部I/Oを持たない純粋関数群とし、Places応答のモックで単体テスト可能にする。

### コスト対策
- Places各SKU 月1万回まで無料枠 + **KVキャッシュ命中**で実コール激減（競合調査は人気店が繰り返し引かれるため効果大）
- **Field Mask**で必要項目のみ取得し安価なSKUに収める
- Workers 10万req/日・KV・Turnstile は無料枠
- メールは Resend 無料枠（月3千通）。送信は詳細版希望者のみ

## 4. 評価ロジック

Googleローカル検索の公式3軸（関連性 / 距離 / 知名度）に沿う。

### 4-1. 整備評価（100点満点・客観）
Places API (New) で取得可能なフィールドのみで算出。業種別に配点の重みを切替（飲食は写真比重↑など）。

| カテゴリ | 配点 | 判定に使うPlacesフィールド（例） |
|---|---|---|
| 基本情報(NAP) | 20 | displayName / formattedAddress / nationalPhoneNumber / websiteUri / businessStatus |
| カテゴリ設定 | 15 | primaryType / types（主・副カテゴリの有無と業種マッチ） |
| 口コミ | 25 | userRatingCount / rating / reviews[].publishTime（直近90日の新着率）/ 星のばらつき |
| 写真・動画 | 15 | photos[]（点数・上限10で代理測定）/ 動画有無 |
| 営業時間・最新性 | 10 | regularOpeningHours / currentOpeningHours / specialDays |
| 付加情報 | 15 | editorialSummary / priceLevel / 属性 / 予約・メニューリンク / Q&A活用 |

各項目を ◎/○/△/× の4段階判定→重み付けで100点換算。

### 4-2. 検索評価（想定・相対）
実順位は出さず、**知名度指数（Prominence Index）**を算出し、同カテゴリ近隣競合内での**相対ポジション（上位◯%／◯位相当）**として提示。

```
知名度指数 = w1·log(口コミ数) + w2·平均星 + w3·写真点数
           + w4·口コミ新着性 + w5·カテゴリ関連度 + w6·整備スコア
```
- 競合ON時: Nearby/Text Search で同カテゴリ近隣の上位N件を取得 → 各社の指数を計算 → 対象店の順位/偏差を提示。これが**競合調査機能**も兼ねる。
- 競合OFF時: 順位・偏差は出さず、自店の知名度指数（と整備スコア）のみ表示する。
- **必ず「想定値」と明記**（Googleは実順位を保証しないため、誠実かつ規約安全）。

## 5. 画面とフロー

### ① 入力画面（誰でも・メール不要）
- 入力: 事業名（必須）/ 住所・エリア名（必須・どちらの形式も許容）/ 競合比較トグル（任意, OFFで自店のみ）
- Turnstileは不可視でバックグラウンド判定
- 「無料でMEO診断する」ボタン

### ② ローディング演出
- 段階的な「調査している感」の演出（例：「NAP整合性を照合中…」「競合◯件を取得中…」「口コミ傾向を解析中…」）
- 実処理が数秒でも本格感を出す

### ③ 結果画面（無料版・その場表示）
要素順:
1. ヘッダー（事業名 / エリア / 日付）
2. ヒーロー: クレイ調スコア（カウントアップ）＋一言講評
3. カテゴリ別バー（6項目, 伸びるアニメ）
4. 検索評価（想定）＋競合比較リスト（ONのとき / 自店ハイライト）
5. 改善ポイント: **上位3つのみ表示、残りはぼかし＋「＋ほか◯件」**（もったいぶり）
6. 詳細CTA（**控えめなテキストリンク**）/ 最下部にSearchMania誘導をさりげなく

### ④ メール詳細版（リード獲得）
- メールアドレス入力 → 確認コードをメール送信 → **コードを手入力**して解錠
- 内容は無料版と同等＋もったいぶり部分も全開示（整形レポート形式）
- 末尾に「さらに詳しくはこちら」→ SearchManiaへ

## 6. セキュリティ・不正対策
- **Cloudflare Turnstile**: 全診断リクエストでトークン検証。AIエージェント/ヘッドレス自動化を不可視ブロック
- **メール確認コード**: 詳細版解錠時に数字コードを手入力（人間性担保＋リード品質チェック）
- **レート制限**: IP単位＋日次の上限（KVカウンタ）。超過時はやんわり制限
- Places APIキーはWorker側に秘匿（フロントに露出しない）

## 7. ビジュアル方針
- **B＋H 組み合わせ**: 流体グラデーション背景 ×すりガラス(グラスモーフィズム)のカード × ぷっくりクレイ(クレイモーフィズム)のスコア＆ボタン
- **淡い青系グラデーションを多めに**
- アニメーション多め・派手め（スコアのカウントアップ、バーの伸び、クレイの浮遊、グラデの流動）
- 詳細レポートCTAは目立たせない

## 8. データソース注意
- 真の検索順位・AI検索での表示は無料・低コストでは提供困難なため、検索評価は「想定（相対指数）」として提供する。
- 表示はGoogleの規約に抵触しない範囲（実順位の断定をしない）に留める。

## 9. オープン事項（実装時に詰める）
- 業種別の配点重みテーブルの具体値（全業種対応・初期は汎用＋主要業種いくつか）
- ローディング演出の最小表示時間
- KVキャッシュTTLの最終値（7〜30日のどこか）
- メール詳細版レポートのフォーマット詳細
- レート制限のしきい値
